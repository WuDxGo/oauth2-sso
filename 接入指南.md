## 1. 概述

### 1.1 架构说明

```
┌─────────────────┐     ┌──────────────────┐     ┌─────────────────┐
│   业务系统 A     │     │  认证服务中心     │     │   业务系统 B     │
│  (client_id=A)  │────▶│  (端口 8080)      │◀────│  (client_id=B)  │
│                 │     │                  │     │                 │
└─────────────────┘     └──────────────────┘     └─────────────────┘
                               │
                               ▼
                        ┌─────────────┐
                        │   MySQL     │
                        │   Redis     │
                        └─────────────┘
```

### 1.2 支持的授权模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| `authorization_code` | 授权码模式 | Web 应用（最安全） |
| `password` | 密码模式 | 可信应用、内部系统 |
| `client_credentials` | 客户端凭证模式 | 服务间调用 |
| `refresh_token` | 刷新 Token | Token 续期 |

### 1.3 环境要求

| 组件 | 版本要求 |
|------|----------|
| Java | 17+ |
| MySQL | 8.0+ |
| Redis | 6.0+ |
| Node.js | 18+ |

### 1.4 技术栈

| 分类 | 技术 |
|------|------|
| **后端** | Spring Boot 3.5.11, Spring Cloud 2025.0.1, Spring Security 6.4.2, Spring Authorization Server |
| **前端** | Vue 3.5, TypeScript 5, Vite 6, Element Plus 2 |
| **数据库** | MySQL 8.0+, Redis |
| **认证** | OAuth2.1, JWT |

## 2. 快速开始

### 2.1 接入流程

```
步骤 1: 初始化数据库
    ↓
步骤 2: 启动认证服务
    ↓
步骤 3: 配置客户端
    ↓
步骤 4: 集成 OAuth2 客户端
    ↓
步骤 5: 测试联调
```

### 2.2 步骤详解

#### 步骤 1：初始化数据库

```bash
# 执行 SQL 脚本
mysql -u root -p < init.sql
```

#### 步骤 2：启动认证服务

**修改配置文件** `oauth-server/src/main/resources/application.yml`：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3307/oauth2_sso?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai
    username: root
    password: your_password
  data:
    redis:
      host: localhost
      port: 6379
```

**启动服务**：

```bash
cd oauth-server
mvn spring-boot:run
```

或使用 IDEA 启动 `OAuth2ServerApplication` 类。

#### 步骤 3：启动前端（可选，用于管理客户端）

```bash
cd frontend
npm install
npm run dev
```

访问：**http://localhost:3000**

#### 步骤 4：配置客户端

登录认证中心管理后台（http://localhost:3000），进入**客户端管理**页面，添加新客户端。

#### 步骤 5：集成 OAuth2 客户端

根据您的技术栈选择对应的接入方式（见 [接入示例](#4-接入示例)）。

#### 步骤 6：测试联调

使用 Postman 或浏览器测试登录流程。

## 3. 配置客户端

### 3.1 客户端配置参数

| 参数 | 必填 | 说明 | 示例 |
|------|------|------|------|
| `client_id` | 是 | 客户端唯一标识 | `order-service` |
| `client_name` | 是 | 客户端显示名称 | `订单服务` |
| `client_secret` | 是 | 客户端密钥 | `order-secret` |
| 认证方式 | 是 | `client_secret_basic` / `client_secret_post` | `client_secret_basic` |
| 授权模式 | 是 | 授权码/密码/客户端凭证 | `authorization_code,password` |
| 重定向 URI | 授权码模式必填 | 回调地址 | `http://localhost:8081/callback` |
| 授权范围 | 是 | 权限范围 | `openid,profile,read,write` |
| 需要授权同意 | 否 | 是否需用户确认 | `否` |

### 3.2 典型配置示例

#### 示例 1：Web 应用（授权码模式 + 自动授权）

```yaml
client_id: web-app
client_name: Web 应用
client_secret: web-app-secret
认证方式：client_secret_basic
授权模式：authorization_code,refresh_token
重定向 URI: http://localhost:3000/oauth2/callback
授权范围：openid,profile,read,write
需要授权同意：否
```

> **说明**：当 `需要授权同意` 设置为 `否` 时，用户登录成功后会立即回调到接入方，无需再次确认。

#### 示例 2：微服务（客户端凭证模式）

```yaml
client_id: order-service
client_name: 订单服务
client_secret: order-service-secret
认证方式：client_secret_basic
授权模式：client_credentials
授权范围：read,write
需要授权同意：否
```

#### 示例 3：移动 App（密码模式）

```yaml
client_id: mobile-app
client_name: 移动客户端
client_secret: mobile-app-secret
认证方式：client_secret_basic
授权模式：password,refresh_token
授权范围：openid,profile,read
需要授权同意：否
```

## 4. 接入示例

### 4.1 授权码模式完整流程（推荐）

#### 4.1.1 流程说明

```
1. 用户访问接入方网站 → 发现未登录
2. 重定向到 SSO 认证中心登录页
3. 用户在 SSO 页面登录
4. SSO 回调到接入方（带 code）
5. 接入方后端用 code 换 token
6. 接入方后端用 token 换用户信息
7. 创建本地会话，登录完成
```

#### 4.1.2 前端配置（Vue 示例）

**路由守卫 - 拦截未登录：**

```javascript
// router/index.js
router.beforeEach((to, from, next) => {
  const token = localStorage.getItem('access_token');

  if (!token && to.meta.requiresAuth) {
    // 构建 SSO 登录 URL
    const ssoLoginUrl = `http://localhost:8080/oauth2/authorize?` +
      `client_id=web-app&` +
      `redirect_uri=${encodeURIComponent('http://localhost:3000/oauth2/callback')}&` +
      `response_type=code&` +
      `scope=openid%20profile%20read%20write&` +
      `state=${encodeURIComponent(crypto.randomUUID())}`;

    // 跳转到 SSO 登录页
    window.location.href = ssoLoginUrl;
  } else {
    next();
  }
});
```

**回调页面 - 处理授权码：**

```vue
<!-- views/OAuth2Callback.vue -->
<template>
  <div class="callback-page">
    <el-result icon="loading" title="登录中" />
  </div>
</template>

<script setup>
import { onMounted } from 'vue';
import { useRouter, useRoute } from 'vue-router';
import { ElMessage } from 'element-plus';

const router = useRouter();
const route = useRoute();

onMounted(async () => {
  const code = route.query.code;
  const state = route.query.state;

  if (!code) {
    ElMessage.error('授权失败');
    router.push('/login');
    return;
  }

  try {
    // 调用后端，用 code 换 token
    const response = await fetch('/api/auth/callback', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ code, state })
    });

    if (!response.ok) {
      throw new Error('登录失败');
    }

    const result = await response.json();

    // 保存 token
    localStorage.setItem('access_token', result.data.access_token);
    localStorage.setItem('expires_in', result.data.expires_in);

    ElMessage.success('登录成功');
    router.push('/dashboard');
  } catch (error) {
    console.error('登录失败:', error);
    ElMessage.error('登录失败，请重试');
    router.push('/login');
  }
});
</script>
```

#### 4.1.3 后端配置（Spring Boot 示例）

**添加依赖：**

```xml
<dependencies>
    <!-- OAuth2 客户端 -->
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-oauth2-client</artifactId>
    </dependency>

    <!-- OAuth2 资源服务器 -->
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-oauth2-resource-server</artifactId>
    </dependency>
</dependencies>
```

**配置文件：**

```yaml
# application.yml
spring:
  security:
    oauth2:
      client:
        registration:
          sso:
            client-id: web-app
            client-secret: web-app-secret
            authorization-grant-type: authorization_code
            redirect-uri: http://localhost:8081/login/oauth2/code/sso
            scope: openid,profile,read,write
            client-name: SSO 认证中心
        provider:
          sso:
            issuer-uri: http://localhost:8080
            authorization-uri: http://localhost:8080/oauth2/authorize
            user-info-uri: http://localhost:8080/api/users/me
            jwk-set-uri: http://localhost:8080/oauth2/jwks
```

**安全配置：**

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .authorizeHttpRequests(authorize -> authorize
                .requestMatchers("/", "/public/**").permitAll()
                .anyRequest().authenticated()
            )
            .oauth2Login(oauth2 -> oauth2
                .loginPage("/login")
                .defaultSuccessUrl("/dashboard")
                .failureUrl("/login?error=true")
            )
            .oauth2ResourceServer(oauth2 -> oauth2
                .jwt(jwt -> jwt.jwtAuthenticationConverter(jwtAuthenticationConverter()))
            );
        return http.build();
    }

    @Bean
    public JwtAuthenticationConverter jwtAuthenticationConverter() {
        JwtGrantedAuthoritiesConverter grantedAuthoritiesConverter = new JwtGrantedAuthoritiesConverter();
        grantedAuthoritiesConverter.setAuthorityPrefix("ROLE_");
        grantedAuthoritiesConverter.setAuthoritiesClaimName("authorities");

        JwtAuthenticationConverter jwtAuthenticationConverter = new JwtAuthenticationConverter();
        jwtAuthenticationConverter.setJwtGrantedAuthoritiesConverter(grantedAuthoritiesConverter);
        return jwtAuthenticationConverter;
    }
}
```

### 4.3 服务间调用（客户端凭证模式）

#### 4.3.1 获取 Token

```java
@Service
public class OAuth2ClientService {

    @Value("${oauth2.client.id}")
    private String clientId;

    @Value("${oauth2.client.secret}")
    private String clientSecret;

    @Value("${oauth2.server.token-uri}")
    private String tokenUri;

    public String getClientCredentialsToken() {
        // 构建 Basic Auth 头
        String auth = clientId + ":" + clientSecret;
        String encodedAuth = Base64.getEncoder().encodeToString(auth.getBytes());

        // 构建请求
        HttpHeaders headers = new HttpHeaders();
        headers.setContentType(MediaType.APPLICATION_FORM_URLENCODED);
        headers.set("Authorization", "Basic " + encodedAuth);

        MultiValueMap<String, String> body = new LinkedMultiValueMap<>();
        body.add("grant_type", "client_credentials");
        body.add("scope", "read write");

        HttpEntity<MultiValueMap<String, String>> request = new HttpEntity<>(body, headers);

        // 发送请求
        RestTemplate restTemplate = new RestTemplate();
        ResponseEntity<Map> response = restTemplate.postForEntity(tokenUri, request, Map.class);

        return (String) response.getBody().get("access_token");
    }
}
```

#### 4.3.2 携带 Token 调用服务

```java
@Service
public class OrderServiceClient {

    @Autowired
    private OAuth2ClientService oauth2ClientService;

    public Order getOrderById(Long orderId) {
        String token = oauth2ClientService.getClientCredentialsToken();

        HttpHeaders headers = new HttpHeaders();
        headers.set("Authorization", "Bearer " + token);
        headers.setContentType(MediaType.APPLICATION_JSON);

        HttpEntity<Void> request = new HttpEntity<>(headers);

        RestTemplate restTemplate = new RestTemplate();
        ResponseEntity<Order> response = restTemplate.exchange(
            "http://order-service/api/orders/" + orderId,
            HttpMethod.GET,
            request,
            Order.class
        );

        return response.getBody();
    }
}
```

## 5. API 接口

### 5.1 OAuth2 端点

| 端点 | 方法 | 说明 |
|------|------|------|
| `/oauth2/authorize` | GET | 授权端点（授权码模式） |
| `/oauth2/jwks` | GET | JWK 集合（验证 JWT） |
| `/oauth2/revoke` | POST | Token 吊销端点 |
| `/oauth2/introspect` | POST | Token  introspection 端点 |
| `/.well-known/openid-configuration` | GET | OIDC 发现端点 |
| `/userinfo` | GET | 用户信息端点（OIDC） |

### 5.2 业务端点

| 端点 | 方法 | 说明 | 认证 |
|------|------|------|------|
| `/api/login` | POST | 登录接口（返回 JWT） | 无需 |
| `/api/users/me` | GET | 获取当前用户信息 | 需要 |
| `/api/clients` | GET/POST | 获取/创建客户端列表 | 需要 |
| `/api/clients/{id}` | GET/PUT/DELETE | 客户端详情/更新/删除 | 需要 |

### 5.3 登录接口示例

**请求：**

## 6. 常见问题

### 6.1 认证失败

**问题：** 401 Unauthorized

**可能原因：**
- `client_id` 或 `client_secret` 错误
- Token 已过期
- 请求头格式错误

**解决方法：**
```bash
# 检查 Basic Auth 格式
echo -n "client_id:client_secret" | base64
# 确保 Authorization 头为：Basic <base64 编码>
```

### 6.2 跨域问题

**问题：** 前端无法访问认证服务

**解决方法：** 认证服务已配置 CORS，如需添加新的允许源，修改 `DefaultSecurityConfig.java`：

```java
configuration.setAllowedOrigins(Arrays.asList(
    "http://localhost:3000",
    "http://localhost:8081",
    "https://your-domain.com"  // 添加生产环境域名
));
```

### 6.3 Token 验证失败

**问题：** 业务服务无法验证 Token

**解决方法：** 确保业务服务配置了正确的 JWK URI：

```yaml
spring:
  security:
    oauth2:
      resourceserver:
        jwt:
          jwk-set-uri: http://localhost:8080/oauth2/jwks
```

### 6.4 重定向 URI 不匹配

**问题：** `redirect_uri` 与配置不一致

**解决方法：** 确保请求中的 `redirect_uri` 与客户端配置**完全一致**（包括协议、端口、路径）

### 6.5 数据库连接失败

**问题：** 无法连接到 MySQL

**解决方法：**
1. 确认 MySQL 已启动
2. 检查 `application.yml` 中的数据库连接配置
3. 确认数据库 `oauth2_sso` 已创建

### 6.6 登录失败

**问题：** 使用测试账号登录失败

**解决方法：**
1. 确认已执行 `init.sql` 初始化数据库
2. 应用首次启动会自动创建默认用户和密码（密码：123456）
3. 查看控制台日志确认用户已正确创建
